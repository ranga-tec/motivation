Executive Summary

NeurAlsEdge is pleased to present this comprehensive proposal for the implementation of a Prosthetic & Orthotic Management System (POMS) tailored specifically for Exceed Prosthetics & Orthotics. This enterprise-grade solution will revolutionize your multi-location clinic operations across Sri Lanka, Philippines, and Cambodia.

Understanding Your Challenges

Exceed P&O operates as a social enterprise providing life-changing prosthetic and orthotic services across three countries. Managing patient records, clinical workflows, episode tracking, and multi-location operations requires a robust, secure, and integrated system. Current manual or fragmented systems can lead to:

  • Inefficient patient data management across multiple locations
  • Difficulty tracking treatment episodes from assessment to delivery
  • Limited reporting capabilities for clinical and administrative insights
  • Challenges in document management and secure file storage
  • Inconsistent patient numbering and location-based tracking
Our Solution: POMS

POMS is a production-ready, enterprise-grade Patient & Episode Management System built with ASP.NET Core 8.0 and designed specifically for prosthetic and orthotic clinics. The system provides comprehensive patient tracking, clinical workflow management, and true multi-location support with location-specific patient numbering.

Key Differentiator: Unlike generic medical software, POMS is purpose-built for P&O clinics with specialized modules for prosthetic episodes (amputation types, levels, sides), orthotic episodes (body regions, orthosis types, main problems), and spinal orthosis treatments (pathological conditions, orthotic designs).

System Features & Capabilities

1. Patient Management Module
Patient Registration & Tracking

Comprehensive patient lifecycle management from registration to treatment completion with mandatory guardian information for patient safety and family engagement.

  • Complete Patient Profiles: Personal information, contact details, medical history, and conditions
  • Mandatory Guardian Information: Guardian name, relationship, address, and contact numbers (required for all patients)
  • Multi-Location Patient Numbering: Automatic generation with format {CenterCode}-{Year}-{Sequence}
    • Sri Lanka - Ragama: SL-RGM-2025-0001, SL-RGM-2025-0002...
    • Philippines - Quezon City: PH-QC-2025-0001, PH-QC-2025-0002...
    • Cambodia - Phnom Penh: KH-PP-2025-0001, KH-PP-2025-0002...
  • Advanced Search & Filtering: By patient number, name, national ID, center, district, registration date range
  • Patient Document Management: Upload and securely store ID cards, medical reports, insurance documents (PDF, JPG, PNG, DOCX)
  • Complete Audit Trail: Track who created, updated, or modified patient records with timestamps

Episode Management Module

2. Episode Management Module
Specialized Episode Types

Prosthetic Episodes: For artificial limb treatments

  • Amputation types: Below/Above Knee, Below/Above Elbow, Partial Hand/Foot
  • Amputation level and side tracking (Left/Right/Bilateral)
  • Reason for amputation: Disease, Trauma, Vascular, Diabetic, Cancer, Congenital
  • Date of amputation tracking

Orthotic Episodes: For braces and support devices

  • Main problem description (free text for detailed documentation)
  • Body region: Upper Limb, Lower Limb, Spine, Other
  • Orthosis type from catalog: AFO, KAFO, TLSO, LSO, WHO, and custom types
  • Reason for problem documentation
  • Side tracking (Left/Right/Bilateral/N/A)

Spinal Orthosis Episodes: For back and spine support

  • Pathological condition documentation (e.g., scoliosis with curve degree)
  • Orthotic design specification (e.g., TLSO custom-molded rigid brace)
  • No side field (spine is central)
Episode Workflow Features
  • Integrated Workflow: Assessment, Fitting, and Delivery can be filled during episode creation
  • Episode Status Management: Open/Closed status with automatic duration tracking
  • Episode Documents: Separate document management per episode (assessments, X-rays, fitting photos)
  • Episode History: Complete timeline view of all clinical activities

Repair Module

  • Detailed repair documentation for warranty management
  • Repair history tracking for quality assurance

Document Management System

4. Document Management System

Dual Document Storage:

  • Patient Documents: General files (ID cards, medical reports, insurance) stored at patient level
  • Episode Documents: Treatment-specific files (assessments, X-rays, fitting photos) stored per episode

Security Features:

  • File type validation (PDF, JPG, JPEG, PNG, DOCX only)
  • File size limits (10MB per file, configurable)
  • Secure upload/download with authentication
  • Organized storage: {RootPath}/patients/{PatientNumber}/{Year}/{Month}/
  • Metadata tracking: Title, upload date, uploaded by, file size, remarks
  • Soft delete with audit trail (data never permanently deleted)

Security & Technical Architecture

7. Security & Compliance Features
Authentication & Authorization
  • ASP.NET Core Identity: Industry-standard user management with secure password hashing (PBKDF2)
  • Account Lockout: Automatic lockout after 5 failed login attempts
  • Password Policy: Requires uppercase, lowercase, digit, special character, minimum 8 characters
  • Role-Based Access Control: Three user roles with detailed permission matrix
  • Session Management: Secure cookies with HttpOnly and Secure flags, 30-minute sliding expiration
User Roles & Permissions
Role Access Level Key Permissions
ADMIN Full System Access User management, system configuration, all reports, audit logs, device catalog, location management
CLINICIAN Clinical Operations Patient registration, episode management, clinical documentation (assessment, fitting, delivery), clinical reports, document management
DATA_ENTRY Basic Data Entry Patient registration, episode creation, document upload, view dashboard (limited access)
Audit Trail & Data Protection
  • Automatic Audit Logging: All entities track CreatedBy, CreatedAt, UpdatedBy, UpdatedAt, DeletedBy, DeletedAt
  • Change History: Complete tracking of all system changes with before/after values
  • Soft Delete: Data never permanently deleted, ensuring compliance and recovery capability
  • Query Filters: Deleted records automatically excluded from normal queries
Data Encryption & Security
  • HTTPS Enforced: All connections use SSL/TLS 1.2+ encryption
  • Database Encryption: Support for Transparent Data Encryption (TDE) on SQL Server
  • File System Encryption: Recommended BitLocker (Windows) or LUKS (Linux) for file storage
  • SQL Injection Prevention: Entity Framework parameterized queries throughout
  • XSS Protection: Automatic HTML encoding in all views

File Upload Security
  • File size validation (configurable maximum)
  • File extension whitelist (PDF, JPG, JPEG, PNG, DOCX only)
  • Content type validation
  • Filename sanitization to prevent directory traversal
  • Isolated storage directories per patient/episode
  • Access control on all file downloads
